workflow

workflow-проще говоря это алгоритм. Задача workflow довольна проста-составить последовательность действий и шагов. Это очень помогает в сфере бизнеса

как он работает:

  • как алгоритм,он начинает с самого начала.Определяют задачи от саммого трудного к самому легкому. Определяются задачи для достижения цели.

  • указывает ктор за что будет ответственнен.

  • Определяют и делят ресурсы и инструменты.

  • После всего вышеперечисленного,сотрудники/участники команды начинают свою работу.

  • Во время процесса важно отслеживать работу каждого сотрудника и успешно закончить свой проект.

git merge

git merge-помогает сохранить данные одной ветки в другую. Если случайным образом у вас удалилась ветка develop,но перед этим вы успели выполнить команду git merge,данные ветки develop будут сохранены,но в другой ветке.

git merge нужен для:

  • Сотрудничество: Позволяет нескольким разработчикам работать над разными функциями или исправлениями, а затем объединять свои изменения.
  • Управление версиями: Помогает сохранить историю изменений, позволяя отслеживать, кто и когда вносил изменения.
  • Разделение задач: Вы можете работать над новой функцией в отдельной ветке, не мешая основной ветку, а затем объединить её, когда будете готовы.

work rebase

Git rebase — это команда в Git, которая позволяет переносить изменения из одной ветки на другую, изменяя историю коммитов. Она работает, переписывая историю, чтобы сделать её более линейной, что упрощает понимание изменений. Rebase нужен для упрощения слияния веток и устранения лишних коммитов, чтобы история проекта была чистой и понятной.

как он работает:

  • выбираете ветку,куда собираетесь перенести изменения

  • Git берёт ваши коммиты из текущей ветки и применяет их один за другим на выбранную базу, как если бы они были созданы заново.

  • Если возникают конфликты, Git приостанавливает процесс, и вам нужно вручную их решить. После этого вы продолжаете процесс с помощью команды git rebase --continue.

  • В результате у вас получается линейная история, где ваши изменения идут сразу после последнего коммита в целевой ветке.

Git fetch

git fetch — это команда в Git, которая позволяет загрузить обновления из удалённого репозитория в ваш локальный репозиторий, но не сливает их с вашей текущей веткой.

как он работает:

  • Получение изменений: git fetch проверяет удалённый репозиторий на наличие новых коммитов, веток или тегов.

  • Обновление ссылок: Команда обновляет локальные ссылки на удалённые ветки, но не изменяет ваши локальные файлы или рабочую ветку.

отличие между git rebase от git merge

git merge сохраняет историю веток, создавая дополнительный коммит, в то время как git rebase переписывает историю, создавая более чистый и линейный вид..

отличие между git fetch от git pull

git fetch просто получает изменения, а git pull сразу их применяет. Это делает pull более удобным, но fetch предоставляет больше контроля, позволяя сначала проверить изменения.
